1. Core Pages for SaaS Dashboards

Overview / Executive Summary

This is the landing page. Even internal dashboards nearly always have one.

  • Total customers
  • Active customers
  • New customers in last 7 / 30 days
  • Churned customers in last 7 / 30 days
  • MRR or ARR if you have billing
  • Simple trend lines for customers and revenue
Why it matters:
  • Quick health check without drilling down
  • Useful even if metrics are rough
Easy to implement:
  • Aggregate counts
  • Daily or monthly grouped queries
  • Line charts from time series tables

2. Customers List

This is the most used page in practice.

Common table columns:
  • Name
  • Email or account owner
  • Plan / tier
  • Status (trial, active, cancelled, paused)
  • Created date
  • Last active date
  • MRR
Useful extras:
  • Search
  • Filters by plan, status, date
  • Click through to customer detail
Easy to implement:
  • Django admin patterns translate well
  • Plain ORM queries
  • Server side pagination

3. Customer Detail Page

SaaS teams live in this view.

Typical sections:
  • Account info
  • Current plan and billing status
  • Usage metrics
  • Activity timeline
  • Notes or internal tags
Why it matters:
  • Support, sales, product all need this
  • Explains why someone churned or upgraded
Easy to implement:
  • Normalised models + related queries
  • No heavy aggregation required

2. Growth and Retention Metrics

4. Retention and Cohort Analysis

This is where SaaS companies start feeling “grown up”.

Common views:
  • Cohort table by signup month
  • % active after 7, 30, 90 days
  • Retention curves
Why it matters:
  • Shows product stickiness
  • More useful than raw churn alone
Implementation difficulty:
  • Medium
  • Requires grouping users by signup date and checking activity flags
  • Tip: start with binary active yes or no. Do not overcomplicate early.

5. Activation Funnel

Very common, very revealing.

Example funnel:
  • Signed up
  • Completed onboarding
  • Performed core action
  • Became “active” user
Components:
  • Funnel counts
  • Conversion percentages
  • Drop off points
Easy to implement:
  • Yes, if you define clear boolean events
  • Harder if your product actions are fuzzy

3. Usage and Behaviour Metrics

6. Product Usage

This is where you spot future churn early.

Typical metrics:
  • DAU / WAU / MAU
  • Feature usage counts
  • Actions per user
  • Time since last activity
Useful visualisations:
  • Line chart of active users
  • Bar chart of top used features
  • Heatmap of usage by day
Implementation difficulty:
  • Easy if you already log events
  • Hard if you don’t log anything yet
  • If you log even one table like: user_id | event_type | created_at you unlock a lot.

7. Customer Health Score

Very normal internally, rarely shown externally.

Inputs:
  • Last activity
  • Usage frequency
  • Support tickets
  • Billing status
Output:
  • Green / amber / red
  • Numeric score 0 to 100
Why it matters:
  • Flags accounts to save before they churn
  • Helps prioritise outreach
Easy to implement:
  • Yes, if you keep it simple
  • Weighted sum of a few metrics

4. Revenue and Billing (if applicable)

8. Revenue Metrics

If you charge money, this becomes essential.

Common metrics:
  • MRR
  • ARR
  • ARPU
  • Expansion vs contraction revenue
  • Trial to paid conversion
Graphs:
  • MRR growth over time
  • Revenue by plan
  • New vs lost MRR
Implementation difficulty:
  • Easy if Stripe or similar is your source of truth
  • Hard if billing logic is custom and messy

9. Plans and Pricing Analysis

Often overlooked early on.

Useful views:
  • Customers per plan
  • Revenue per plan
  • Churn by plan
  • Upgrade / downgrade flows
Easy to implement:
  • Simple group by queries
  • Very high insight to effort ratio

5. Support and Operations Metrics

10. Support Signals

Even lightweight tracking helps.

Metrics:
  • Tickets per customer
  • Time to first response
  • Open vs resolved
  • Tickets before churn
Why it matters:
  • Support load predicts churn
  • Shows which customers are “expensive”
Easy to implement:
  • Easy if you own support data
  • Otherwise integrate later

6. Easy Wins in Django

High value, low effort additions:

  • Saved filters (views per role)
  • Export CSV
  • Date range picker
  • Compare two time ranges
  • Annotations on charts (feature launch, price change)
  • Role based views (admin vs support vs sales)

7. What to Skip Early

Most teams regret overbuilding these early on:

  • Predictive churn ML
  • Perfect attribution models
  • Complex LTV calculations
  • Real time dashboards
  • Over granular funnels

You can add those later when you actually feel the pain.

Suggested Minimal but Solid Structure

Pages:
  • Dashboard
  • Customers list
  • Customer detail
  • Usage
  • Retention
  • Revenue (if applicable)
Shared components:
  • Date range selector
  • Metric cards
  • Line chart
  • Table with filters
  • Export button
